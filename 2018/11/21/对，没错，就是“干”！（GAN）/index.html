对，没错，就是“干”！（GAN）

发表于 2018-11-21 | 分类于 algorithm |

GAN（Generative Adversarial Nets）于14年由Ian J. Goodfellow提出，虽然在理论上看起来还是相当的漂亮，但是在实际的训练上却没那么的漂亮，直到17年W-GAN的提出，以及仅仅三个月后W-GAN-GP的提出才弥补了这一缺陷，而且理论上更加的漂亮，似乎已经无懈可击，但是最近Google的一项研究表示，不管是什么GAN的变种，其表现和原始GAN实际上都相差无几，对于庞大的GAN家族，无奈😂😂😂……

本文只介绍原始GAN的思想、推导以及缺陷，后续内容有时间在继续推出。

Basic idea of GAN

GAN的目标函数（价值函数）：

上述公式中，D是判别器，G是生成器，从判别器的角度看，它希望自己尽可能区分真实样本和虚假样本，因此希望$D(x)$尽可能大，$D(G(z))$尽可能小，即$V(D,G)$尽可能大。从生成器G的角度看，它希望自己生成的样本尽可能骗过D，也就是希望$D(G(z))$尽可能大，即$V(D,G)$尽可能小，两个模型相互对抗/博弈，最后达到全局最优。（$P_{data}(x)$为真实样本的数据分布，$P_{z}(z)$为原始噪音的分布，$G(z)$为生成器/映射函数，即为噪音z经过生成器后生成的新的数据，其所有数据聚集起来服从$P_G$分布，即z经过生成器后的数据分布，学习的目标就是要让$P_{data}(x)$与$P_G$尽可能一样，即让$P_G$去拟合$P_{data}(x)$这个数据分布。）

证明：对上述目标函数进行最优化求解有唯一解$G^{*}$，并且该唯一解满足$P_G=P_{data}$

还原即得$ D^{}(x) = \frac{P_{data}(x)}{P_{data}(x) + P_G(x)} \Rightarrow 0< D^{}(x) < 1 $当算法收敛得到最优解$P_{data}(x) = P_G(x)$时，$D^(x) \equiv \frac{1}{2}$。即对于最优生成器生成的$G(z) = x$，最优判别器$D^_{G}(x)= \frac{1}{2}$，即该判别器已经无法判别该样本是来自$P_{data}(x)$还是来自$P_{G}(x)$了，即认为x来自$P_{data}(x)$和$P_{G}(x)$的概率都是$\frac{1}{2}$.
求出了最优判别器$D^$，则进行第二步，给定$D^$，$ \min \limits_{G} V(G, D^*) $：

即第二步就是在$\min P_{data}(x)$与$P_{G}(x)$之间的JS散度，也就是在拉近$P_{data}(x)$与$P_{G}(x)$的距离。这就证明了优化上述目标函数能够得到唯一解$G^*$，且该唯一解满足$P_{data}(x) = P_{G}(x)$。

（1）这里的证明是基于$E_{z \sim P_{z}(z)}log(1 - D(G(z))) = E_{x \sim P_G}(log(1-D(x)))$的，该等式源于测度论中的RN定理，但是用在这里的证明并不严谨，只不过大家都这么说；
（2）JS散度：

$0 < JS < log2$，也用于衡量两个分布之间的距离，同KL散度，但是JS散度相较于KL散度具有对称性。

对上述过程的通俗化说明：

第一步：G为随机给定的一个分布，记为$G_0$，其产生的图片为$G_{0}(z)$，首先$ \max \limits_{D} E_{x \sim P_{data}}logD(x) + E_{z \sim P_{z}(z)}log(1 - D(G_{0}(z)))$。即对于来自真实样本的x，使得$D(x)$尽可能的大，对于来自$P_{z}(z)$分布中的样本z，尽量使$D(G(z))$变小，这样根据原始$G_{0}$就产生了最优的$D_{G_{0}}^$；
第二步：给定$D_{G_{0}}^$，计算$\min \limits_{G} V(G_{0}, D_{G_{0}}^)$，即在给定$D_{G_{0}}^$的情况下，更新$G_{0}$的参数，使得$D(G(z))$变大，也就是让$G_{0}$生成的图片尽量真实，从而让$D_{G_{0}}^$给出更高的分数。其等价于让$1 - D(G(z))$变小，由于第一项$E_{x \sim P_{data}}logD(x)$与更新$G_{0}$的参数毫无关系，因此可直接忽略。因此就变成了$\min \limits_{G} E_{z \sim P_{z}(z)}log(1 - D(G_{0}(z))) = G_{1}$，更新完成后即得到了第一代$G_{1}$；
第三步：给定$G_{1}$，$\max \limits_{D} V(G_{1}, D_{G_{0}}^)) = D_{G_{1}}^)$；
第四步：给定$D_{G_{1}}^$， $\min \limits_{G}V(G_{1}, D_{G_{1}}^)) = G_2$；
……循环往复……
注意：用梯度上升法（GA）找最优的$D^$时需要更新参数多次直至找到最优的判别器，而在更新G时，一般只需更新一次，因为更新多次的话就不能保证JS下降了。

GAN中discriminator与generator结合的优势：

discriminator比较容易感知component与component之间的相关性，而generator不容易感知component与component之间的相关性，因此每个component由generator独立生成，即generator在decoder一张图片时，神经网络在输出时并不能考虑到单元与单元之间的相关性，所以将discriminator和generator组合成一个神经网络，其前半部分结构为generator，生成完图片后再让discriminator对其评分，当discriminator感知到pixel与pixel之间有空白pixel的时候，尽管此时与原图只相差一个pixel，但是discriminator依然会给其很低的分数，因此generator和discriminator组合成一个NN的时候，就会增强整个结构的robust性（能用更浅的NN结构产生与很深的VAE结构相差不大甚至更好的效果）。

原始GAN存在的一些缺陷

1、在最优$D^$条件下，实际上$P_{data}$和$P_G$不会有任何相交的部分，所以此时$P_{data}$和$P_{G}$不管是相距很远还是相距很近，他们的JS散度都等于$log2$，表现在图像中即为下图（1），即在最优$D^$条件下JS散度的图像如（1）所示：

此时$\frac{\partial JS(P_{data}(x)||P_{G}(x))}{\partial \theta_g} = 0 $（斜率为0），无法给G提供梯度，G的loss降不下来（JS散度），梯度消失。这时在训练中就会表现出G的loss一直是$log2$，而D的loss一直接近0.
在不是最优$D^$条件下，此时依然将其近似为$P_{data}(x)$和$P_{G}(x)$的JS散度，但此时$P_{G}$和$P_{data}$有了相交的部分，此时JS散度的图像可能表现为（2），且$JS(P_{data}(x)||P_{G}(x) < log2$，所以在理想$D^$条件下，$P_{G}$和$P_{data}$的JS散度反映的就是G的loss，我们能通过这个loss看到训练的进程，此时在更新$\theta_g$时也能提供大于0的梯度，使得G的loss得到下降，但这种情况仅仅出现在理想的$D^$情况下。
实际上原始GAN如果要收敛的很好，$D^$要满足两个条件，首先$D^$需要训练到最优，因为只有最优条件下在给定$D^$后才能将loss近似为JS散度；第二$D^*$又不能训练到最优，只有不是最优的，JS散度的图像才可能是上图中的（2），这样才能稳步的拉近两个分布之间的距离，否则就会梯度消失，在实际操作中，会“折衷”满足这两个条件，但是这个“折衷”的度却很难把握，因此原始GAN的训练才会很困难；
2、对于实际操作过程中的G的loss，$L_{G} = E_{x \sim P_{G}}[-logD(x)]$也非常不合理，由：

而$E_{x \sim P_{data}}[logD^(x)] + E_{x \sim P_{G}}[log(1- D^(x))] = 2JS(P_{data} || P_{G}) - 2log2$，
即$E_{x \sim P_{data}}[logD^(x)] - 2JS(P_{data} || P_{G}) + 2log2 = -E_{x \sim P_{G}}[log(1- D^(x))]$，
$\Rightarrow E_{x \sim P_{G}}[-logD^(x)] = KL(P_{G}||P_{data}) - 2JS(P_{data} || P_{G}) + E_{x \sim P_{data}}[logD^(x)] + 2log2 $

对于KLD来说，由于其不是一个对称的量，以$KL (P_G||P_{data})$为例：

也就是说，当G“没能生成真实样本”的时候，产生的loss会很小，对于优化器来说，它是“喜欢”的，因为不用费太大劲就能把它降下去，而当G“生成了不真实的样本”后，产生的loss就会很大，对于优化器来说，它很“不喜欢”，因为它需要费很大的劲才能把它降下去，此时对G来说，它就宁可去多生成一些重复但是很“安全”的样本，也不愿意去生成更加多样化的样本，因为那样一不小心就会产生很大的loss，得不偿失，这种现象就是collapse mode（生成的样本中有很多重复样本且缺乏多样性）。
